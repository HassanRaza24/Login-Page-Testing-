# Test Cases Documentation

## Project
Login Module Testing

## Overview
This document contains manual test cases created for testing the Login functionality of the application.

The purpose of these test cases is to verify that the login feature works correctly with different input conditions and meets the expected requirements.

## Testing Type
- Manual Testing
- Functional Testing

## Test Case Structure

| Field | Description |
|---|---|
| Test Case ID | Unique identifier of each test case |
| Scenario ID | Related test scenario reference |
| Test Case Title | Description of the functionality being tested |
| Precondition | Conditions required before executing the test |
| Test Steps | Steps performed to execute the test |
| Test Data | Input values used during testing |
| Expected Result | Expected behavior of the application |
| Status | Test execution result (Pass/Fail) |

## Sample Test Case

**Test Case ID:** TC001

**Scenario ID:** TS001

**Title:** Verify login with valid credentials

**Precondition:**
User account should exist.

**Test Steps:**
1. Open login page.
2. Enter valid email.
3. Enter valid password.
4. Click Login button.

**Test Data:**
- Email: test@gmail.com
- Password: 123456

**Expected Result:**
User should be successfully logged in and redirected to the dashboard.

**Status:**
Pass

Test cases were designed based on functional requirements and cover positive and negative testing scenarios.
